# Spec: Добавление торговых инструментов в избранное

**Status:** active
**Plane:** 4211ae9c-67f7-45e7-a1f6-3ae42e1d7a1b
**Created:** 2026-06-30

## Problem

Авторизованному пользователю AtlasQuant нужен персональный список избранных валютных фьючерсов MOEX, чтобы быстро возвращаться к нужным инструментам. Сейчас каталог `/instruments` общий для всех; нет способа сохранить `secid` в профиле пользователя.

## Acceptance Criteria

- [x] AC-1: Модель `FavoriteInstrument` (`user_id`, `secid`, `shortname`, `asset_code`); валидации presence; уникальность `[user_id, secid]`; `User has_many :favorite_instruments`.
- [x] AC-2: `Favorites::Add.call(user:, secid:)` проверяет, что `secid` есть в `Moex::CurrencyFutures::List.call`, сохраняет snapshot `shortname`/`asset_code`; при неизвестном `secid` возвращает `Result` с `success?: false` и сообщением об ошибке; при дубликате — idempotent success (существующая запись).
- [x] AC-3: `Favorites::Remove.call(user:, secid:)` удаляет запись; если записи нет — `success?: false` с сообщением «not in favorites».
- [x] AC-4: Маршруты (auth required): `GET /favorites` — список избранного; `POST /instruments/:secid/favorite` — добавить; `DELETE /instruments/:secid/favorite` — удалить. Гость редиректится на `new_session_path` с alert.
- [x] AC-5: UI: на `GET /instruments/:secid` кнопка Add/Remove favorite (Turbo-friendly `button_to`); на `GET /favorites` карточки с ссылками на график; в header nav ссылка «Favorites» только для logged-in пользователя.
- [x] AC-6: Minitest: model test; unit tests для `Favorites::Add` и `Favorites::Remove` (stub MOEX list); controller tests для create/destroy/index (auth + happy/error paths).
- [x] AC-7: `bin/ci` завершается с exit code 0.

## Out of Scope

- Полный CRUD модели `Instrument` в PostgreSQL
- Real-time MOEX WebSocket / paid data
- Сортировка/теги/заметки к избранному
- Funding calculator, basis analytics на странице favorites
- RSpec-миграция (Minitest per текущий стек)
- Redis, OAuth/Devise

## Constraints

- MVP boundaries из AGENTS.md; доменная логика в `app/services/favorites/`
- Инструменты по-прежнему on-demand из MOEX ISS; в БД хранится только `secid` + display snapshot
- Auth: `has_secure_password`, `require_login` для favorites
- Security: strong params не нужны для secid из route; CSRF через Rails forms
- UI: ERB + Tailwind; без новых npm-зависимостей
