# Spec: Add trading instruments to favorites

**Status:** active
**Plane:** 4211ae9c-67f7-45e7-a1f6-3ae42e1d7a1b
**Created:** 2026-06-29

## Problem

Авторизованному пользователю AtlasQuant нужно сохранять валютные фьючерсы MOEX в персональный список избранного, чтобы быстро возвращаться к нужным инструментам без повторного поиска в общем каталоге.

## Acceptance Criteria

- [x] AC-1: Модель `Favorite` (`user_id`, `secid`) с уникальным индексом `[user_id, secid]` и валидациями presence; `User has_many :favorites`.
- [x] AC-2: `Favorites::Add.call(user:, secid:)` создаёт запись, если `secid` есть в `Moex::CurrencyFutures::List.call`; при дубликате возвращает `success?: false` с ошибкой uniqueness; при неизвестном `secid` — `success?: false` с ошибкой invalid secid.
- [x] AC-3: `Favorites::Remove.call(user:, secid:)` удаляет запись; если избранного нет — `success?: false` с понятной ошибкой.
- [x] AC-4: Маршруты `GET /favorites`, `POST /favorites`, `DELETE /favorites/:secid` защищены `require_login`; неавторизованный пользователь редиректится на `/session/new` с flash alert.
- [x] AC-5: На `/instruments` и `/instruments/:secid` для авторизованного пользователя отображается кнопка добавления/удаления из избранного; после действия — redirect back с flash notice.
- [x] AC-6: `GET /favorites` показывает таблицу избранных инструментов пользователя (ticker, name, asset, ссылка на chart); пустой список — сообщение «No favorites yet».
- [x] AC-7: Minitest: model test `Favorite`; unit tests для `Favorites::Add` и `Favorites::Remove` (stub MOEX list); controller tests для auth gate, create, destroy, index.
- [x] AC-8: `bin/ci` завершается с exit code 0.

## Out of Scope

- Полноценная модель `Instrument` и CRUD полей инструмента
- Real-time MOEX WebSocket, funding calculator
- Избранное для неавторизованных пользователей (localStorage/cookies)
- RSpec-миграция (остаёмся на Minitest до отдельной задачи)
- Redis, OAuth/Devise, мультибиржевость

## Constraints

- MVP boundaries из AGENTS.md; доменная логика в `app/services/favorites/`
- Идентификатор инструмента — MOEX `secid` (string), без дублирования MOEX-данных в БД кроме `secid`
- Auth: существующий `has_secure_password` + session; strong params для `secid`
- UI: ERB + Tailwind; Turbo-friendly `button_to` для POST/DELETE
- Security: только владелец может управлять своими favorites (`current_user.favorites`)
