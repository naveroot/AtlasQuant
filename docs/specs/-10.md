# Spec: Миграция тестов Minitest → RSpec + SimpleCov

**Status:** active
**Plane:** 97e8b639-a65c-43fb-bda9-81f072e006b0
**Created:** 2026-06-30

## Problem

Целевой стек MVP — RSpec + SimpleCov (AGENTS.md), но в `test/` остаются 11 Minitest-файлов и нет RSpec. Технический долг блокирует наращивание фич с гарантией качества и единым CI-контуром.

## Acceptance Criteria

- [x] AC-1: `bundle exec rspec` завершается с exit code 0 (все перенесённые тесты зелёные).
- [x] AC-2: SimpleCov настроен с порогом ≥80% покрытия для групп `app/models`, `app/services`, `app/controllers`; при нарушении порога RSpec завершается с ненулевым exit code.
- [x] AC-3: `config/ci.rb` запускает `bundle exec rspec` вместо `bin/rails test`.
- [x] AC-4: `.github/workflows/ci.yml` job `test` запускает RSpec (`bundle exec rspec`) вместо Minitest; job `system-test` использует RSpec system specs (или эквивалент `bundle exec rspec spec/system`).
- [x] AC-5: Каталог `test/` удалён из репозитория; Minitest больше не используется как test runner.
- [x] AC-6: `bin/ci` завершается с exit code 0.

## Out of Scope

- Миграция orchestrator TypeScript-тестов
- Добавление новых фич приложения (auth CRUD, funding calculator и т.д.)
- Покрытие helpers, jobs, mailers сверх переноса существующих Minitest-тестов
- Подключение rubocop-rspec или factory_bot

## Constraints

- MVP boundaries из AGENTS.md
- Минимальный diff: перенос существующих тестов 1:1, без рефакторинга production-кода
- Security: не коммитить секреты; coverage report в `.gitignore`
- Conventional Commits для PR
