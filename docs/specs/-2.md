# Spec: Deploy to TimeWeb (CI/CD)

**Status:** active
**Plane:** fd416378-1ab6-4bed-a15b-08abe53d2ebb
**Created:** 2026-06-29

## Problem

AtlasQuant уже контейнеризован (Dockerfile, Kamal), но автоматический деплой на TimeWeb не настроен. Нужен CI/CD: проверки на PR/push и автоматический выкат на production при merge в `main`, без секретов в репозитории.

## Acceptance Criteria

- [x] AC-1: В `.github/workflows/deploy.yml` job `deploy` запускается только после успешного CI workflow `CI` на ветке `main` и выполняет `bin/kamal deploy`.
- [x] AC-2: `config/deploy.yml` содержит конфигурацию Kamal для TimeWeb VPS: `ghcr.io` registry, placeholder хоста `__TWC_VPS_HOST__`, PostgreSQL accessory, proxy SSL и env для Solid (без raw credentials).
- [x] AC-3: Файл `.kamal/secrets.cd` читает секреты только из переменных окружения (`RAILS_MASTER_KEY`, `KAMAL_REGISTRY_PASSWORD`, `ATLAS_QUANT_DATABASE_PASSWORD`); в репозитории нет реальных ключей.
- [x] AC-4: В корне есть `docker-compose.yml` для TimeWeb Cloud Apps: сервис `web` с портом хоста ≠ 80/443, без запрещённых директив (`volumes`, `privileged` и т.д.), подключение к БД через `DATABASE_URL`.
- [x] AC-5: `config/database.yml` (production) и `config/environments/production.rb` поддерживают деплой: `DB_HOST` / `DATABASE_URL`, `assume_ssl`/`force_ssl` при `KAMAL_PROXY_SSL=true`.
- [x] AC-6: `docs/plans/-2.md` содержит таблицу required GitHub Secrets/Variables и шаги первичной настройки TimeWeb VPS (SSH, ghcr, accessory db).
- [x] AC-7: `bin/ci` завершается с exit code 0.

## Out of Scope

- Создание VPS/Apps в панели TimeWeb (ручная настройка инфраструктуры)
- TimeWeb Kubernetes / Container Registry (используем ghcr.io + Kamal)
- Staging environment и multi-destination Kamal
- Миграция тестов на RSpec
- Интеграция с биржевыми API, Redis, OAuth

## Constraints

- MVP boundaries из AGENTS.md: Kamal + Docker, PostgreSQL (Solid Cache/Queue/Cable), без Redis
- Секреты только через GitHub Secrets и Kamal secrets; не коммитить `config/master.key`
- Минимальный diff: не рефакторить auth и доменную логику
- Deploy workflow не запускается на PR (только post-CI на `main`)
