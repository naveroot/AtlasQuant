# Spec: MOEX currency futures quotes and chart

**Status:** active
**Plane:** 4d6ec914-1594-4603-8499-e530cafd1596
**Created:** 2026-06-29

## Problem

Пользователю AtlasQuant нужно видеть исторические котировки валютных фьючерсов. Источник данных — публичный MOEX ISS API (бесплатные delayed-данные). Нужно определить endpoint'ы, загрузить историю и отобразить график цены закрытия.

## Acceptance Criteria

- [x] AC-1: `app/services/moex/client.rb` выполняет GET к `https://iss.moex.com/iss/...`, парсит ISS JSON (columns + data) в массив хэшей; при HTTP ≠ 200 или timeout возвращает `Moex::Client::Error`.
- [x] AC-2: `app/services/moex/currency_futures/list.rb` возвращает список валютных фьючерсов FORTS с `asset_code` в `Si`, `Eu`, `Cn`, `ED`, `USDRUBTOM`, `EURRUBTOM`, `CNYRUBTOM`; каждый элемент содержит `secid`, `shortname`, `asset_code`.
- [x] AC-3: `app/services/moex/currency_futures/historical_candles.rb` по `secid`, `from`, `till` (ISO date) запрашивает daily candles (`interval=24`) и возвращает массив `{ traded_at:, open:, high:, low:, close:, volume: }`; при пустом ответе — пустой массив.
- [x] AC-4: Маршрут `GET /instruments` — таблица валютных фьючерсов со ссылками; `GET /instruments/:secid` — страница с line chart close-цен за последние 30 календарных дней (форма выбора `from`/`till`, default last 30 days).
- [x] AC-5: При ошибке MOEX или неизвестном `secid` контроллер показывает flash alert и редирект на `/instruments` (show) или пустую таблицу с alert (index).
- [x] AC-6: Minitest: unit-тесты для `Moex::Client`, `Moex::CurrencyFutures::List`, `Moex::CurrencyFutures::HistoricalCandles` (stub HTTP); controller test для `InstrumentsController#show` с stub сервиса.
- [x] AC-7: `bin/ci` завершается с exit code 0.

## Out of Scope

- Real-time WebSocket / paid MOEX subscription
- CRUD персонального списка инструментов (`Instrument` model, auth-gated watchlist)
- Funding rate calculator
- Мультибиржевость, Redis, OAuth/Devise
- Фоновая синхронизация в PostgreSQL (данные запрашиваются on-demand с Solid Cache)

## Constraints

- MVP boundaries из AGENTS.md; доменная логика в `app/services/moex/`
- HTTP через `Net::HTTP`, без новых HTTP-гемов
- График: ERB + Tailwind + Stimulus + Chart.js через importmap (без npm/webpack)
- Кэш ответов MOEX: `Rails.cache` (Solid Cache), TTL 15 минут
- Security: strong params для date range; без секретов MOEX (публичный API)
- Тесты: Minitest (RSpec — целевой стек, миграция отдельно)
