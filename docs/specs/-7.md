# Spec: Instrument charts display and UX

**Status:** active
**Plane:** 67c5d11f-a60f-4628-ba0b-5e2e89016748
**Created:** 2026-06-29

## Problem

На странице инструмента (`GET /instruments/:secid`) график котировок не отображается из‑за некорректной передачи JSON в HTML data-атрибуты Stimulus. Пользователю нужен наглядный финансовый график (свечи OHLC) без таблицы с числовыми данными; интерфейс списка и страницы инструмента должен быть удобнее.

## Acceptance Criteria

- [x] AC-1: На `GET /instruments/:secid` при наличии свечей отображается интерактивный candlestick-график OHLC за выбранный период (библиотека Lightweight Charts через importmap + Stimulus).
- [x] AC-2: JSON данных свечей передаётся в Stimulus через HTML-экранированные data-атрибуты (`tag` helper); график инициализируется без ошибок парсинга.
- [x] AC-3: Таблица OHLC с числовыми колонками на странице инструмента удалена; вместо неё — summary-карточки (последняя цена, изменение за период, min/max, объём).
- [x] AC-4: Страница списка `GET /instruments` — карточки инструментов со ссылкой на график (без табличного списка цифр).
- [x] AC-5: На странице инструмента — быстрые пресеты периода (7 / 30 / 90 дней) + форма `from`/`till`; attribution TradingView в footer блока графика.
- [x] AC-6: Minitest: controller test подтверждает наличие `price-chart` controller, candlestick data attribute и отсутствие OHLC `<table>`; `bin/ci` exit 0.

## Out of Scope

- Real-time MOEX WebSocket / paid data
- CRUD персонального watchlist
- Funding calculator
- Миграция на RSpec
- npm/webpack bundler

## Constraints

- MVP boundaries из AGENTS.md
- UI: ERB + Tailwind + Stimulus; chart library via importmap (Lightweight Charts standalone ESM)
- Доменная логика MOEX без изменений (сервисы `app/services/moex/`)
- Security: без новых секретов; date params как сейчас
