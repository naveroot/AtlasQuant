# Spec: Заполнить README.md

**Status:** active

## Problem

Корневой `README.md` содержит только шаблон Rails и не описывает AtlasQuant: назначение, стек, локальную разработку и CI. Новым разработчикам и агентам нужен единый onboarding-документ, согласованный с `AGENTS.md`.

## Acceptance Criteria

- [ ] AC-1: `README.md` содержит раздел «О проекте» с назначением AtlasQuant (отслеживание валютных фьючерсов, funding rate для perpetual) — текст согласован с `AGENTS.md`.
- [ ] AC-2: `README.md` содержит раздел «Требования» с версиями из `mise.toml`: Ruby 3.2.11, PostgreSQL 16, Node 22; указан инструмент Mise.
- [ ] AC-3: `README.md` содержит раздел «Быстрый старт» с командами `mise install`, `mise exec -- bin/setup`, `mise exec -- bin/dev` и URL dev-сервера (`http://localhost:3000`).
- [ ] AC-4: `README.md` содержит раздел «Тестирование и CI» с командами `bin/rails test` и `bin/ci` и кратким перечислением проверок CI (rubocop, bundler-audit, brakeman, tests).
- [ ] AC-5: `README.md` содержит раздел «Стек» с Rails 8.1, PostgreSQL, Solid Cache/Queue/Cable (без Redis), Tailwind CSS, Hotwire (Turbo + Stimulus).
- [ ] AC-6: `README.md` содержит раздел «Деплой» со ссылкой на `docs/plans/-2.md` (Kamal + TimeWeb); без raw secrets и без дублирования полной таблицы секретов.
- [ ] AC-7: `bin/ci` завершается с exit code 0.

## Out of Scope

- Перевод `AGENTS.md` или других docs целиком в README
- Документация agent-pipeline / Plane orchestrator (остаётся в `docs/`)
- RSpec migration guide (целевой стек, сейчас Minitest)
- API-документация и OpenAPI
- Badges CI/CD в README (можно добавить позже)

## Constraints

- MVP boundaries из AGENTS.md: без Redis, OAuth, real-time exchange APIs
- Не коммитить секреты, `.env`, `config/master.key`
- Минимальный diff: только `README.md` и SDD-артефакты задачи
- Язык README: русский (как AGENTS.md и остальная проектная документация)
