# Spec: Agent clarification in Plane tasks

**Status:** active
**Plane:** 2f3b9a5e-43dd-4255-b798-e4430bd78d99
**Created:** 2026-06-30

## Problem

SDD-агенты (Supercode, Cloud Agent) при нехватке или двусмысности информации в brief, spec или кодовой базе либо додумывают решение, либо останавливаются без обратной связи человеку. Нужен единый протокол: агент публикует структурированный запрос уточнения **комментарием в Plane work item**, переводит задачу в **Blocked** и ждёт ответа в том же треде — без ухода в локальные заметки без синхронизации.

## Acceptance Criteria

- [x] AC-1: `docs/agent-pipeline/agent-clarification.md` описывает триггеры (когда спрашивать vs продолжать), формат комментария (`[Needs Info]`, роль, нумерованные вопросы, блок «Assumed if no reply»), переход в Blocked и возобновление после ответа человека.
- [x] AC-2: `bash .supercode/workflows/atlasquant/scripts/agent-clarification.sh <role> <issue_id> <question1> [question2…]` POST HTML-комментарий через Plane REST и PATCH state → `blocked`; использует `PLANE_AGENT_API_KEY` с fallback на `PLANE_API_KEY`; exit 0 при успехе, 1 при ошибке API или отсутствии env.
- [x] AC-3: `plane-mcp-context.sh` (режим `clarify`) выводит инструкции для Plane MCP: `create_work_item_comment` + `update_work_item` → blocked с тем же HTML-шаблоном, что в AC-2.
- [x] AC-4: Промпты Supercode SDD (`swe-architect.yml`, `swe-spec-review.yml`, `swe-grounding.yml`, `swe-implement.yml`, `swe-pipeline.yml`) содержат правило: при нехватке информации — **не угадывать**; опубликовать запрос в Plane (MCP или `agent-clarification.sh`) и остановиться до ответа.
- [x] AC-5: `.orchestrator/src/build-prompt.ts` включает секцию «Needs Info protocol» с тем же правилом для Cloud Agent; `.orchestrator/src/plane-clarification.ts` экспортирует `formatClarificationComment(role, questions, assumed?)`.
- [x] AC-6: Unit-тест `plane-clarification.test.ts` проверяет HTML-формат (роль, список вопросов, optional assumed block); `npm test` в `.orchestrator` exit 0.
- [x] AC-7: Ralph Loop templates (`PROMPT.md`, `active-context.md`, `session-handoff.md`) маршрутизируют open questions → Plane comment + Blocked, не только `.agent-run/`.
- [x] AC-8: `bin/ci` завершается с exit code 0.

## Out of Scope

- Автоматический парсинг ответов человека в Plane (resume по комментарию — ручной перевод в Agent Ready)
- Новый Plane workflow state кроме существующего Blocked
- Изменения Rails-приложения AtlasQuant
- Webhook «ответ получен → auto-resume agent»
- Создание отдельного Plane-пользователя (см. spec #5 — отдельная задача)

## Constraints

- MVP boundaries из AGENTS.md: только agent pipeline / docs
- Секреты только через `.env`; не коммитить API-ключи
- HTML-комментарии Plane (как в существующем pipeline)
- Обратная совместимость: без `PLANE_AGENT_API_KEY` работает `PLANE_API_KEY`
- Минимальный diff: не рефакторить unrelated orchestrator logic
