# Spec: Продуктовая аналитика и канал обратной связи

**Status:** active
**Plane:** 293f15d4-52cf-487b-a6b5-6d4aee2c482e
**Created:** 2026-06-30

## Problem

Для валидации MVP и измерения KPI (activation, retention) нужна продуктовая аналитика и канал сбора обратной связи. Сейчас метрик нет: нельзя построить воронку регистрации и понять, используют ли пользователи ключевые экраны. Нужно подключить согласованный инструмент аналитики, разметить ключевые события и дать пользователю способ оставить feedback из UI.

## Acceptance Criteria

- [x] AC-1: При заданном `POSTHOG_API_KEY` PostHog SDK инициализируется в layout (`application.html.erb`); без ключа аналитика no-op (не ломает UI и тесты).
- [x] AC-2: Событие `signup` отправляется server-side при успешной регистрации (`RegistrationsController#create` → `Analytics::TrackEvent`).
- [x] AC-3: На `GET /instruments/:secid` отправляются client-side события `view_instrument` и `view_basis` с property `secid`.
- [x] AC-4: Модуль `Analytics::TrackEvent` поддерживает событие `add_to_watchlist` (константа + публичный метод); unit-тест подтверждает отправку payload в PostHog Capture API.
- [x] AC-5: В PostHog можно построить воронку `signup` → `view_instrument` как proxy activation (документировано в plan); оба события имеют общий `distinct_id` = `user.id`.
- [x] AC-6: Пользователь может отправить обратную связь из UI: ссылка «Feedback» в header → `GET /feedback/new` → форма → `POST /feedback` с сохранением в БД и flash notice.
- [x] AC-7: Minitest покрывает `Analytics::TrackEvent` (enabled/disabled, signup payload) и `FeedbacksController` (create success/validation); `bin/ci` exit 0.

## Out of Scope

- Self-hosted PostHog / Plausible deployment
- CRUD персонального watchlist (событие `add_to_watchlist` только инструментировано)
- Реализация `BasisCalculator` и отдельной страницы basis
- PostHog dashboards / alerts configuration в UI PostHog
- Email-уведомления о feedback
- Миграция Minitest → RSpec

## Constraints

- MVP boundaries из AGENTS.md: без Redis, OAuth, real-time exchange APIs
- Аналитика: **PostHog Cloud (SaaS)** — project API key через ENV (`POSTHOG_API_KEY`, optional `POSTHOG_HOST`)
- Секреты только через ENV / Kamal secrets; не коммитить ключи
- UI: ERB + Tailwind; Stimulus для client-side tracking
- Доменная логика аналитики в `app/services/analytics/`
- Не отправлять пароли и полные PII в analytics properties
