# Spec: Plane agent account (отдельная учётная запись для агентов)

**Status:** active
**Plane:** ec0b624a-d3cd-4f0f-9140-ecd074574532
**Created:** 2026-06-29

## Problem

Комментарии и обновления состояний в Plane от pipeline (orchestrator, Supercode scripts, cloud agent) публикуются от имени владельца `PLANE_API_KEY` — обычно это человек-администратор. Нужна отдельная учётная запись агента в Plane, чтобы автоматические действия были видны под именем/ролью агента, а не администратора.

## Acceptance Criteria

- [x] AC-1: В `.orchestrator/.env.example` и `.supercode/workflows/atlasquant/.env.example` документирована переменная `PLANE_AGENT_API_KEY` (API-токен пользователя-агента в Plane); `PLANE_API_KEY` остаётся для admin/setup операций.
- [x] AC-2: Файл `docs/agent-pipeline/plane-agent-account.md` описывает пошаговое создание пользователя Plane (display name «Atlas SWE Agent»), добавление в workspace/project и генерацию API-токена.
- [x] AC-3: Orchestrator (`.orchestrator/src/poll-plane.ts`, `start-agent.ts`) и bash-скрипт `update-plane-state.sh` используют `PLANE_AGENT_API_KEY` (fallback на `PLANE_API_KEY`) для PATCH состояния и POST комментариев.
- [x] AC-4: Комментарии pipeline содержат роль агента в HTML (например `<strong>[Orchestrator]</strong>`, `[Cloud Agent]`, `[Spec Review]`) — роль определяется контекстом вызова, не именем администратора.
- [x] AC-5: Скрипт `npm run verify:agent` (в `.orchestrator`) вызывает `GET /api/v1/users/me/` с `PLANE_AGENT_API_KEY` и выводит `display_name` / email; exit code 0 при успехе, 1 при отсутствии или невалидном ключе.
- [x] AC-6: `bin/ci` завершается с exit code 0.

## Out of Scope

- Создание пользователя Plane через API без admin-доступа (ручной шаг в Plane UI)
- Несколько отдельных Plane-пользователей на каждую SDD-роль (Architect, Implement и т.д.) — одна учётная запись агента с ролью в тексте комментария
- Изменения Rails-приложения AtlasQuant (User, auth)
- OAuth / Service Token Plane (используем user-level API key агента)

## Constraints

- MVP boundaries из AGENTS.md: только инфраструктура agent pipeline
- Секреты только через `.env` / GitHub Secrets; не коммитить реальные API-ключи
- Обратная совместимость: при отсутствии `PLANE_AGENT_API_KEY` pipeline продолжает работать с `PLANE_API_KEY`
- Минимальный diff: не рефакторить unrelated orchestrator logic
