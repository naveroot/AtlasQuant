### Shared VPS с Plane (reverse proxy)

Plane уже слушает **:80/:443** (Caddy). AtlasQuant **не** использует Kamal proxy:

| Компонент | Маршрут |
|-----------|---------|
| Plane | `plane.alfapulse.ru` → Caddy → plane-* containers |
| AtlasQuant | `TWC_APP_HOST` → Caddy → `172.17.0.1:8080` → Kamal app |
| PostgreSQL (accessory) | `atlas_quant-db` в Docker network `atlas_quant-net`; `127.0.0.1:5433` на хосте |

Caddy-конфиг: `deploy/plane-shared/`. Установка на сервере:

```bash
ATLAS_APP_HOST=atlas.alfapulse.ru ./deploy/plane-shared/install.sh
```

DNS: A-запись `TWC_APP_HOST` → **тот же IP**, что у `plane.alfapulse.ru` (`89.223.122.237`).

### TimeWeb VPS bootstrap (ручно, один раз)

1. VPS **plane** (`89.223.122.237`) — Docker уже есть.
2. SSH-ключ в GitHub Secret `SSH_PRIVATE_KEY` или `KAMAL_SSH_PRIVATE_KEY`.
3. Порты 22, 80, 443 открыты.
4. GitHub secrets/variables (см. выше).
5. **Kamal Setup** (workflow_dispatch) → поднимает app на `:8080` и регистрирует маршрут в Caddy.
6. Merge в `main` → CI → **Deploy**. Для ручного повтора можно запустить workflow **Deploy** через `workflow_dispatch`.
