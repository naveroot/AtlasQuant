## Required GitHub Configuration

### Secrets (Settings → Secrets and variables → Actions)

| Secret | Назначение |
|--------|------------|
| `SSH_PRIVATE_KEY` | Private key для SSH на TimeWeb VPS |
| `RAILS_MASTER_KEY` | Rails credentials (`config/master.key`) |
| `ATLAS_QUANT_DATABASE_PASSWORD` | Пароль PostgreSQL (accessory + app) |

### Variables

| Variable | Пример | Назначение |
|----------|--------|------------|
| `TWC_VPS_HOST` | `185.x.x.x` | Публичный IP TimeWeb VPS |
| `TWC_APP_HOST` | `atlas.example.com` | Домен для Kamal proxy SSL |

### TimeWeb Cloud Apps (Docker Compose)

| Variable | Назначение |
|----------|------------|
| `DATABASE_URL` | Primary DB (`atlas_quant_production`) |
| `CACHE_DATABASE_URL` | Solid Cache DB |
| `QUEUE_DATABASE_URL` | Solid Queue DB |
| `CABLE_DATABASE_URL` | Solid Cable DB |
| `RAILS_MASTER_KEY` | Rails credentials |

`KAMAL_REGISTRY_PASSWORD` в deploy workflow = `secrets.GITHUB_TOKEN` (ghcr.io).

### TimeWeb VPS bootstrap (ручно, один раз)

1. Создать VPS в [Timeweb Cloud](https://timeweb.cloud/my/servers) (Ubuntu 24.04, Docker pre-installed или `curl -fsSL https://get.docker.com | sh`).
2. Добавить SSH public key; сохранить private key в `SSH_PRIVATE_KEY`.
3. Открыть порты 22, 80, 443.
4. В GitHub: задать secrets/variables выше.
5. **Первый деплой:** Actions → **Kamal Setup** (workflow_dispatch, один раз) → затем push в `main` → CI green → **Deploy** workflow → `kamal deploy`.
6. Альтернатива без Kamal: TimeWeb Cloud Apps → Docker Compose → подключить репозиторий, задать `DATABASE_URL`, `CACHE_DATABASE_URL`, `QUEUE_DATABASE_URL`, `CABLE_DATABASE_URL` (managed PostgreSQL, отдельные БД для Solid) и `RAILS_MASTER_KEY` в UI.
