# Plan #1: User Auth MVP

Plane issue: `3c26f612-b986-41af-8439-542714b98f0e`

## Scope

User registration, login, logout with `has_secure_password`. Domain logic in services. ERB + Tailwind UI. Minitest coverage. No OAuth, Redis, or real-time exchange APIs.

## Acceptance Criteria

- [x] Enable `bcrypt` gem and add `User` model with `has_secure_password`
- [x] Migration: `users` table with `email` (unique, indexed) and `password_digest`
- [x] `Users::Register` service handles user creation with validation
- [x] `Sessions::Authenticate` service handles credential verification
- [x] `RegistrationsController` with `new`/`create` actions
- [x] `SessionsController` with `new`/`create`/`destroy` actions
- [x] Session-based auth helpers on `ApplicationController` (`current_user`, `logged_in?`, `require_login`)
- [x] Routes for sign up, sign in, sign out
- [x] Tailwind-styled views for registration and login forms
- [x] Layout navigation reflects auth state (sign in / sign up / sign out)
- [x] Tests for `User` model, both services, and both controllers
- [x] `bin/ci` passes (rubocop, brakeman, bundler-audit, tests)
